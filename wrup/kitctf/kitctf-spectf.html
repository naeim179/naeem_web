kitCTF — specCTF Reverse Engineering Writeup

Challenge Description

> We present for our desert a slightly deranged ghost, even though they may be unstable, they are still delicious. Share a meal and a CPU with this ghost and we're sure you'll be satisfied.

The description strongly hints at Spectre speculative execution attacks:

  • "ghost" → Spectre
  • "unstable" → speculative execution
  • "share a CPU" → cache side-channel attacks require sharing CPU resources

At first glance, the challenge appears to require recovering secret data through a Spectre-style side channel.

Understanding main()

The decompiled main():

while (true) {
    pin_core();
    init_attack();
    train();

    sVar2 = strlen(argv[1]);

    if ((sVar2 & 7) != 0)
        break;

    local_14 = 0;

    for (i = 0; i < sVar2 >> 3; i++) {
        local_14 += specte_byte(0x1337, 0x1337);
    }

    if (local_14 == (sVar2 >> 3))
        local_c++;
    else
        local_c--;

    if (local_c > 2) {
        puts("CORRECT");
        exit(0);
    }
}

Important observations:

  1. Input length must be a multiple of 8.
  2. Success depends entirely on specte_byte().
  3. The program repeatedly trains a predictor and measures cache timing.
  4. This is clearly modeled after a Spectre proof-of-concept.

The Red Herring

The intended challenge theme is Spectre.

Functions such as:

get_from_array()

contain:

return arr2[((uint8_t)arr1[idx]) << 9];

This is the classic Spectre gadget:

temp &= probe_array[secret_byte * 512];

and timing functions measure cache hits.

However, the important discovery is that the binary also contains a global array named:

ENC

and a hashing routine:

ulong hashy(ulong x)
{
    ulong y;

    y = (x ^ (x >> 33))
        * 0xf451af975d152cad;

    y ^= (y >> 33);
    y ^= 0xc2ceaade1a351c23;

    return y ^ (y >> 33);
}

Reversing hashy()

This function resembles a MurmurHash-style finalizer.

The key observation:

  • XOR with right shifts is invertible.
  • Multiplication by an odd 64-bit constant is invertible modulo 2⁶⁴.

Therefore the entire function is reversible.

We compute:

INV_C1 = pow(C1, -1, 1 << 64)

and undo operations in reverse order.

Inverting the Hash

MASK = (1 << 64) - 1

def unxor_rshift(y, s):
    x = y
    for _ in range(6):
        x = y ^ (x >> s)
    return x & MASK

C1 = (-0x0bae5068a2ead353) & MASK
C2 = (-0x3d315521e5cae3dd) & MASK

INV_C1 = pow(C1, -1, 1 << 64)

def inv_hash(y):
    y = unxor_rshift(y, 33)
    y ^= C2
    y = unxor_rshift(y, 33)
    y = (y * INV_C1) & MASK
    y = unxor_rshift(y, 33)
    return y

Recovering ENC

The binary stores encrypted chunks inside:

ENC[]

Each entry is a 64-bit value.

Conceptually:

ENC[i] = hashy(flag_chunk[i]);

So recovery becomes:

flag_chunk = inv_hash(ENC[i])

for every entry.

Extracting the Flag

After applying the inverse transformation to all entries of ENC, the plaintext becomes:

Conclusion

Although the challenge is themed around Spectre and speculative execution, the actual solve path is much simpler:

  1. Reverse the binary.
  2. Identify the global encrypted array ENC.
  3. Reverse hashy().
  4. Apply the inverse to every stored 64-bit block.
  5. Recover the flag directly.

The Spectre implementation acts mostly as thematic misdirection, while the real reversing task is recognizing that the hash function is completely invertible.

Lessons Learned

  • Do not let theme functions distract from simpler static data.
  • Odd constants are invertible modulo 2^64.
  • Right-shift XOR finalizers can be inverted iteratively.
