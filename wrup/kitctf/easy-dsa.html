easy-dsa Writeup

Challenge Summary

The service implements an ECDSA-like signing system on curve P-521. We can ask the server to sign chosen messages, but we must forge a valid signature for a recipe that was not signed by the server.

The final flag was:

GPNCTF{MaY8E wE shOuld h4ve hIrED a PrOF3sSI0nAl?}

Vulnerability

P-521 has an order of about 521 bits, so ECDSA nonces should also be about 521 bits.

However, the challenge generated the nonce using SHA-256:

k = sha256(...).digest()

So every nonce satisfies:

k < 2^256

That means the nonce is much smaller than the curve order.

For ECDSA:

s = k^-1 * (z + r*d) mod n

Rearranging:

s*k = z + r*d mod n

where:

  • d is the private key
  • k is the nonce
  • z is the hash of the message
  • (r, s) is the signature
  • n is the curve order

For two signatures:

s0*k0 = z0 + r0*d mod n
si*ki = zi + ri*d mod n

Eliminate d:

ri*s0*k0 - r0*si*ki = ri*z0 - r0*zi mod n

Because all k values are small, this becomes a Hidden Number Problem. We can solve it with LLL/CVP.

Exploit Strategy

  1. Connect to the remote service.
  2. Request the public key.
  3. Ask for several signatures on chosen messages.
  4. Build a lattice from the equations above.
  5. Recover one small nonce k0.
  6. Compute the private key:
d = (s0*k0 - z0) * r0^-1 mod n
  1. Verify that d*G equals the server public key.
  2. Forge a signature for a new recipe.
  3. Submit it to get the flag.

Solver

import socket, ssl, re
from hashlib import sha256
from fpylll import IntegerMatrix, LLL, CVP
from Crypto.PublicKey import ECC

HOST = "grilled-salmon-with-juiced-rosemary-4wow.gpn24.ctf.kitctf.de"
PORT = 443

curve = ECC._curves["p521"] if "p521" in ECC._curves else ECC._curves["P-521"]
n = int(curve.order)
G = curve.G

def H(m):
    return int.from_bytes(sha256(m).digest(), "big")

def recv_until(s, marker):
    data = b""
    while marker not in data:
        data += s.recv(4096)
    return data

def sendline(s, x):
    s.sendall(x + b"\n")

def parse_sig(out):
    r = int(re.search(rb"s1:\s*(0x[0-9a-f]+)", out).group(1), 16)
    ss = int(re.search(rb"s2:\s*(0x[0-9a-f]+)", out).group(1), 16)
    return r, ss

ctx = ssl.create_default_context()
s = ctx.wrap_socket(socket.socket(), server_hostname=HOST)
s.connect((HOST, PORT))

recv_until(s, b"> ")

sendline(s, b"get pkey")
out = recv_until(s, b"> ")
qx = int(re.search(rb"x:\s*([0-9]+)", out).group(1))
qy = int(re.search(rb"y:\s*([0-9]+)", out).group(1))

msgs = [f"recipe{i}".encode() for i in range(8)]
sigs = []

for m in msgs:
    sendline(s, b"sign " + m.hex().encode())
    out = recv_until(s, b"> ")
    sigs.append(parse_sig(out))

mcount = len(msgs)
dim = mcount + (mcount - 1)

S = 1 << 120
rows = []

for j in range(mcount):
    row = [0] * dim
    row[j] = 1

    for eq in range(1, mcount):
        idx = mcount + eq - 1

        r0, s0 = sigs[0]
        ri, si = sigs[eq]

        if j == 0:
            row[idx] = ((ri * s0) % n) * S
        elif j == eq:
            row[idx] = ((-r0 * si) % n) * S

    rows.append(row)

for eq in range(1, mcount):
    row = [0] * dim
    row[mcount + eq - 1] = n * S
    rows.append(row)

target = [0] * dim

for eq in range(1, mcount):
    r0, s0 = sigs[0]
    ri, si = sigs[eq]
    target[mcount + eq - 1] = ((ri * H(msgs[0]) - r0 * H(msgs[eq])) % n) * S

M = IntegerMatrix.from_matrix(rows)
LLL.reduction(M)
v = CVP.closest_vector(M, tuple(target))

k0 = int(v[0])
r0, s0 = sigs[0]
z0 = H(msgs[0])

d = None
for kk in [k0, -k0, k0 % n, (-k0) % n]:
    cand = ((s0 * kk - z0) * pow(r0, -1, n)) % n
    P = cand * G
    if int(P.x) == qx and int(P.y) == qy:
        d = cand
        break

if d is None:
    print("failed")
    exit()

print("[+] private key:", hex(d))

msg = b"give me the flag"
k = 1337
P = k * G
fr = int(P.x) % n
fs = (pow(k, -1, n) * (H(msg) + fr * d)) % n

sendline(s, b"flag please")
recv_until(s, b"recipe")
sendline(s, msg.hex().encode())
recv_until(s, b"s1")
sendline(s, hex(fr).encode())
recv_until(s, b"s2")
sendline(s, hex(fs).encode())

print(s.recv(4096).decode(errors="ignore"))

Takeaway

The challenge looked like normal ECDSA on a strong curve, but the nonce generation was broken. Using a 256-bit nonce on P-521 leaks enough information for a lattice attack. Once one nonce is recovered, the private key follows immediately, allowing arbitrary signature forgery.

Evidence

Clue 01

P-521 expects roughly 521-bit nonces, but SHA-256 bounds k below 2^256.

Clue 02

Small nonces turn the signatures into a Hidden Number Problem.

Clue 03

Recovered d verifies against the public key before forging the final signature.

Lessons Learned

  • Strong curves still fail with weak nonce generation.
  • Nonce size must match the group order security requirements.
  • Always verify recovered keys against the public key before forging.
