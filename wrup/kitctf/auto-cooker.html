Auto Cooker – GPNCTF Reverse Engineering Writeup

Challenge Overview

The challenge provides a binary called Auto Cooker. According to the description, we input a "recipe" (flag), and the machine "cooks" it through several transformations before checking whether the result matches an internal target value.

The goal is to reverse these transformations and recover the original flag.

Initial Analysis

After loading the binary into Ghidra, the main function becomes straightforward.

The program:

  1. Reads user input.
  2. Stores it in a buffer called FOOD.
  3. Applies a sequence of functions:
  • salt()
  • fry()
  • trim()
  • mix()
  • taste()
  1. taste() compares the final result against a hardcoded byte array called DELICIOUS.

Simplified flow:

salt(FOOD);
fry(FOOD);
trim(FOOD);
mix(FOOD);

if (taste(FOOD))
    puts("Good meal!");

Since the comparison is performed after all transformations, we can recover the flag by reversing the operations applied to DELICIOUS.

Step 1 – Analyzing salt()

The first function iterates through the buffer and XORs each byte with 0xAA.

for (i = 0; i < 64; i++)
{
    FOOD[i] ^= 0xAA;
}

This operation is reversible because:

(x ^ 0xAA) ^ 0xAA = x

Step 2 – Analyzing fry()

The next function swaps the upper and lower nibbles of every byte.

FOOD[i] = (FOOD[i] << 4) | (FOOD[i] >> 4);

Example:

0xAB → 0xBA
0x3F → 0xF3

Nibble swapping is its own inverse:

swap(swap(x)) = x

Therefore, reversing it is simply applying the same operation again.

Step 3 – Analyzing trim()

The trim() function modifies only the last few bytes:

for (i = 0x3d; i < 0x40; i++)
{
    FOOD[i] &= 0x0F;
}

This masks off the upper nibble and keeps only the lower 4 bits.

Since information is lost here, the operation is not perfectly reversible.

Fortunately, this only affects the final three bytes of the buffer and does not prevent recovery of the actual flag.

Step 4 – Analyzing mix()

This function reverses the order of the 64-byte buffer.

for (i = 0; i < 64; i++)
{
    TMP[i] = FOOD[63 - i];
}

Example:

ABCDEFGH
↓
HGFEDCBA

Reversing the array again restores the original order.

Step 5 – Extracting the Target

Inside taste() we find the constant array:

unsigned char DELICIOUS[64] = {
    ...
};

This is the value the transformed flag must equal.

To recover the original flag we perform the inverse operations in reverse order:

DELICIOUS
↓
reverse mix
↓
reverse fry
↓
reverse salt
↓
FLAG

Solve Script

target = [
    0x0a,0x0a,0x0a,0x0a,0x7d,0xdd,0x5c,0x4e,
    0x5f,0xf9,0xfc,0x2c,0x9d,0x3e,0xec,0x5f,
    0xd9,0x9d,0x99,0xee,0x8d,0xbe,0x2e,0x5f,
    0x8f,0xfd,0x5e,0x5f,0x8d,0x5c,0xce,0x5f,
    0x3d,0xec,0xbc,0xfc,0x8f,0x5f,0xfc,0x8f,
    0xbe,0x5f,0xff,0x5e,0x3f,0x5f,0x99,0x1c,
    0x3c,0xb9,0x5f,0x6e,0xfe,0xfc,0xce,0x5f,
    0x3e,0x1d,0xce,0xef,0x9e,0x4e,0xaf,0xde
]

# Reverse mix
data = target[::-1]

# Reverse fry (same operation)
data = [((b << 4) & 0xff) | (b >> 4) for b in data]

# Reverse salt
data = [b ^ 0xaa for b in data]

print(bytes(data).decode(errors="ignore"))

Running the script reveals the flag:

after action report

Lessons Learned

  • Invertible transformations should be undone in reverse order.
  • XOR and nibble swaps are common reversible obfuscation primitives.
  • Even lossy-looking code can be harmless if it only touches padding bytes.
