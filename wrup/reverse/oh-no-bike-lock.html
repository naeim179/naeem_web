Walkthrough

Challenge Description

"OH NO!!! You forgot the PIN for your bike lock. Analyze the lock and figure out how to break it."

We are given a Linux executable and need to determine the correct flag by reversing the program's logic.

Step 1 – Initial Reconnaissance

The first step is identifying the file type.

file bike_lock
Output:
ELF 64-bit LSB executable

This tells us that the challenge is a compiled Linux binary and should be analyzed using reverse engineering tools such as:

Ghidra

IDA Pro

Binary Ninja

Radare2

Running the program reveals that it asks for a PIN.

Enter PIN:

This suggests the challenge is centered around recovering or reconstructing the PIN validation logic.

Step 2 – Static Analysis

Loading the binary into Ghidra and locating main() reveals something unusual.

Most beginner reverse-engineering challenges perform a direct comparison such as:

if(strcmp(input, "secret_pin") == 0)

or

if(input == expected)

However, this binary does neither.

Instead, it:

Builds an internal numeric string.

Performs several transformations.

Uses the transformed values as indexes into another string.

This indicates that the flag is generated dynamically.

Step 3 – Hidden Data

Inside the binary there are two important hardcoded strings.

The first is a long lookup table:

AweQbLpMnorTyUioZxcvbnmCQwertyuioTqwFer{TyUiopasRd@fghjknLzXcvbnmdQwerty0u0IopasdMFghjklzxc_VbnmqwerYCvbnZMqw4e2RtyuiopasuFghjklzx%SdfyGhjklzmNbmqwerty}

At first glance it appears random.

The second hidden string consists entirely of digits and is used internally to construct the effective PIN.

Step 4 – Understanding the Algorithm

After decompiling the relevant function, the following behavior becomes clear.

The program does not use the user input directly.

Instead it:

1. Constructs a numeric sequence

Several embedded strings are combined to form a larger digit sequence.

Conceptually:

pin_string = part1 + part2 + part3;

2. Takes Every Fourth Character

The program iterates through the numeric sequence using a stride of four.

Pseudo-code:

for(i = 0; i < len(pin_string); i += 4)

{

selected += pin_string[i];

}

Example:

1234567890123456

^ ^ ^ ^

1 5 9 3

Only these positions are retained.

3. Convert Characters to Digits

Each selected character is converted into its numeric value.

digit = ch - '0';

Result:

1 5 9 3 ...

4. Compute Cumulative Offsets

The digits are not used directly.

Instead the program continuously adds them together.

Pseudo-code:

sum += digit;

index = sum;

Example:

Digits:

1 5 9 3

Cumulative:

1

6

15

18

These cumulative totals become indexes into the lookup table.

Step 5 – Extracting Characters

The lookup string acts as a giant character array.

Example:

flag += lookup[index];

Each cumulative index retrieves one character.

Performing the process repeatedly yields:

b

o

r

o

C

T

F

{
...
}

Eventually the complete flag emerges.

Step 6 – Reconstructing the Flag

Following all cumulative offsets and indexing operations produces:

boroCTF{R@nd00M_YZ42u%ym}

This is the exact string generated by the program.

Why This Works

The challenge tries to hide the flag by:

Storing a seemingly random lookup table.

Using arithmetic rather than direct comparisons.

Generating indexes dynamically.

Avoiding the flag appearing in plaintext anywhere in memory.

However, once the indexing algorithm is understood, recovering the flag becomes straightforward.

The entire protection mechanism ultimately boils down to:

digit sequence

↓

every 4th digit

↓

cumulative sums

↓

lookup-table indexes

↓

flag characters

boroCTF{R@nd00M_YZ42u%ym}

Lessons Learned

This challenge demonstrates a common reverse-engineering technique:

Hide the flag inside a lookup table.

Use arithmetic transformations to generate indexes.

Reconstruct the flag at runtime rather than storing it directly.

When reversing binaries, always look for:

Large suspicious strings in .data

Character indexing operations

Arithmetic involving offsets

Loops that build output character-by-character

These patterns often indicate an encoded or generated flag.
