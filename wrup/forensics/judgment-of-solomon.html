Walkthrough

Challenge

Description: The birth of reconstruction.

Overview

The challenge gives us a file called code. At first glance, it looks like a long text/hex dump rather than a normal image file. The title and description hint that something has to be “reconstructed,” and the “Judgment of Solomon” reference suggests that there may be more than one possible “answer,” but only one is real.

The final object turns out to be a damaged QR code. After reconstructing and repairing it, the QR decodes to the real flag.

Step 1: Inspecting the file

Running file on the provided file shows that it is plain text:

file code
Output:
code: ASCII text, with very long lines

Opening the file shows a huge string made mostly of hex-looking values such as:

FFFFFFFFFFFFFFFFFFFFFFFF...

000000000000...

FF0000...

This strongly suggests image data, because FFFFFF, 000000, and FF0000 are common RGB color values:

FFFFFF = white

000000 = black

FF0000 = red

There was also a suspicious embedded flag-looking string inside the data:

boroCTF{I_C0xL6n"+_d0_it_St11nz_n0w_go}

This was not the real flag. It was inserted into the hex stream as a distraction/corruption. The challenge title makes sense here: we have to judge the real result from the fake one.

Step 2: Cleaning the hex stream

After removing the fake boroCTF{...} text from the long line, the remaining length becomes meaningful.

The cleaned data length is:

26268 characters

Since RGB pixels are represented by 6 hex characters each, and each row ends with 0A, the length factors nicely:

26268 = 66 * (66 * 6 + 2)

That means:

66 rows

66 pixels per row

6 hex characters per pixel

2 extra hex characters per row: 0A

So the file is storing a 66×66 RGB image, with each row ending in 0A.

Step 3: Reconstructing the image

I parsed each row, ignored the trailing 0A, then converted every 6 hex characters into an RGB pixel.

Example solve code:

from PIL import Image
import re
with open("code", "r") as f:
raw = f.read().splitlines()[0]
# Remove the fake/corrupt flag-looking text
raw = re.sub(r"boroCTF\{[^}]+\}", "", raw)
W = H = 66
row_len = W * 6 + 2
assert len(raw) == H * row_len
img = Image.new("RGB", (W, H))
for y in range(H):
row = raw[y * row_len : (y + 1) * row_len]
assert row.endswith("0A")
row = row[:-2]
for x in range(W):
px = row[x * 6 : x * 6 + 6]
r = int(px[0:2], 16)
g = int(px[2:4], 16)
b = int(px[4:6], 16)
img.putpixel((x, y), (r, g, b))
img.save("reconstructed_rgb.png")

The output is a broken-looking QR code. It is not directly scannable yet.

Step 4: Understanding the image

The reconstructed image is 66×66, but QR codes are made of square modules. Since 66 = 33 × 2, each QR module is represented as a 2×2 block of pixels.

So the actual QR code grid is:

33×33 modules

A 33×33 QR code corresponds to QR version 4, because QR sizes follow:

21 + 4 × (version - 1)

For version 4:

21 + 4 × 3 = 33

The RGB image also contains red corruption. Splitting the channels shows that the red channel gives the cleanest QR data. The green/blue channels include more damage.

This is another connection to the title: we have to choose the correct “child”/channel and discard the false or corrupted data.

Step 5: Downsampling to QR modules

I downsampled the image from 66×66 pixels into a 33×33 QR grid by taking each 2×2 block and deciding whether it should be black or white.

I used the red channel and treated dark pixels as black:

from PIL import Image
img = Image.open("reconstructed_rgb.png").convert("RGB")
N = 33
modules = [[False for _ in range(N)] for _ in range(N)]
for y in range(N):
for x in range(N):
vals = []
for dy in range(2):
for dx in range(2):
r, g, b = img.getpixel((2 * x + dx, 2 * y + dy))
vals.append(r)
# Black if most pixels in this 2x2 block are dark in the red channel
modules[y][x] = sum(v < 128 for v in vals) >= 2

At this point, the QR data is mostly there, but some of the required QR structural patterns are damaged.

Step 6: Repairing the QR code

QR codes have predictable structural areas:

Three finder patterns in the top-left, top-right, and bottom-left corners

White separators around those finder patterns

Timing patterns along row 6 and column 6

A fixed dark module for the QR version

These areas can be repaired because their positions and shapes are known.

For a 33×33 QR code, the finder patterns start at:

(0, 0)

(26, 0)

(0, 26)

I repaired them manually in the module grid.

def set_module(x, y, value):
if 0 <= x < N and 0 <= y < N:
modules[y][x] = value
def draw_finder(x0, y0):
# Draw 7x7 finder plus 1-module white separator
for y in range(-1, 8):
for x in range(-1, 8):
X = x0 + x
Y = y0 + y
if not (0 <= X < N and 0 <= Y < N):

continue

# Separator
if x in (-1, 7) or y in (-1, 7):
set_module(X, Y, False)

continue

# Finder pattern:
# black outer border, white inner ring, black 3x3 center
black = (

x == 0 or x == 6 or

y == 0 or y == 6 or

(2 <= x <= 4 and 2 <= y <= 4)

)

set_module(X, Y, black)
draw_finder(0, 0)
draw_finder(26, 0)
draw_finder(0, 26)
# Repair timing patterns
for i in range(8, 25):
value = (i % 2 == 0)
set_module(i, 6, value)
set_module(6, i, value)
# Fixed dark module for version 4 QR
set_module(8, 25, True)

Then I rendered the repaired QR with a quiet zone:

from PIL import Image, ImageDraw
scale = 12
quiet = 4
out_size = (N + quiet * 2) * scale
out = Image.new("L", (out_size, out_size), 255)
draw = ImageDraw.Draw(out)
for y in range(N):
for x in range(N):
if modules[y][x]:
x0 = (x + quiet) * scale
y0 = (y + quiet) * scale
draw.rectangle(
[x0, y0, x0 + scale - 1, y0 + scale - 1],

fill=0

)

out.save("repaired_qr.png")

Step 7: Decoding the QR

After repairing the QR code, it became scannable.

Using OpenCV:

import cv2
img = cv2.imread("repaired_qr.png")
data, points, _ = cv2.QRCodeDetector().detectAndDecode(img)
print(data)
Output:
boroCTF{I_f1%ed_wHat_w4$_br0Ken}

Final Flag

boroCTF{I_f1%ed_wHat_w4$_br0Ken}

Takeaway

The main trick was not to trust the obvious embedded flag. The file was really a corrupted RGB hex dump. Once the fake text was removed, the dimensions revealed a 66×66 image. That image was a doubled 33×33 QR code, and after choosing the correct channel and repairing the QR’s known structural patterns, the real flag decoded cleanly.

Lessons Learned

  • Do not trust the first flag-looking string you see.
  • Dimensions and factors can reveal hidden image structure.
  • Knowing QR fixed patterns makes manual repair possible.
