solution notes

Walkthrough

Challenge Summary

We were given four encrypted ZIP archives:

a_USE.zip

b_JOHN.zip

c_THE.zip

d_RIPPER.zip

The filenames immediately looked suspicious. Reading them in order gives the phrase:

USE JOHN THE RIPPER

This strongly hinted that the intended solution was to use John the Ripper to crack the ZIP passwords.

Initial Extraction Attempt

First, I tried extracting the files normally:

unzip a_USE.zip
unzip b_JOHN.zip
unzip c_THE.zip
unzip d_RIPPER.zip

However, unzip failed with errors like:

need PK compat. v5.1 (can do v4.6)

This suggested that the ZIP files used a newer ZIP encryption method, likely AES encryption, which the default unzip tool could not handle properly.

Instead of using unzip, I switched to 7z for extraction after cracking the passwords.

Converting ZIPs to John Hashes

John the Ripper cannot directly crack ZIP files without first converting them into hash format. For that, I used zip2john.

zip2john a_USE.zip > hashes.txt
zip2john b_JOHN.zip >> hashes.txt
zip2john c_THE.zip >> hashes.txt
zip2john d_RIPPER.zip >> hashes.txt

This created a single hash file containing all four ZIP password hashes.

Cracking with RockYou

Next, I ran John using the common rockyou.txt wordlist:

john hashes.txt --wordlist=/usr/share/wordlists/rockyou.txt

John successfully cracked three passwords:

chips (a_USE.zip/USE.log)

fishandchips (b_JOHN.zip/JOHN.log)

sunchips (c_THE.zip/THE.log)

So the first three ZIP passwords were:

a_USE.zip -> chips

b_JOHN.zip -> fishandchips

c_THE.zip -> sunchips

Extracting the First Three Archives

Since unzip had trouble with the archives, I used 7z:

7z x a_USE.zip -pchips
7z x b_JOHN.zip -pfishandchips
7z x c_THE.zip -psunchips

This extracted three log files:

USE.log

JOHN.log

THE.log

Reading them showed some story/hint text:

cat USE.log
cat JOHN.log
cat THE.log

The logs said:

[2026] INFO PASSWORD INSECURE PLEASE MOVE LOGS
[2026] MESSAGE SYSADMIN "fine I'll move them"
[2026] INFO PLEASE MAKE BETTER PASSWORDS
[2026] SYSADMIN MESSAGE "NO"

This confirmed that the challenge was about weak/reused-style passwords.

Cracking the Final ZIP

The last archive, d_RIPPER.zip, was not cracked during the first run. Since all cracked passwords were related to chips, I created a smaller targeted wordlist from rockyou.txt:

grep -i "chip" /usr/share/wordlists/rockyou.txt > chip_words.txt

Then I generated a hash file for only the final ZIP:

zip2john d_RIPPER.zip > ripper_hash.txt

I ran John again, this time using the filtered wordlist with rules:

john ripper_hash.txt --wordlist=chip_words.txt --rules

This successfully cracked the final password:

chip! (d_RIPPER.zip/RIPPER.log)

So the final password was:

chip!

Extracting the Final Archive

Because the password contains an exclamation mark, I quoted it to prevent shell interpretation issues:

7z x d_RIPPER.zip -p'chip!'

This extracted:

RIPPER.log

Reading the file:

cat RIPPER.log

gave:

[2026] ALL PASSWORDS HAVE BEEN BREACHED
[2026] SYSADMIN MESSAGE "boroCTF{L@_R11pP3r;}"

boroCTF{L@_R11pP3r;}

Key Takeaways

The filenames themselves formed the hint:

USE JOHN THE RIPPER

The challenge required converting encrypted ZIP files into John-compatible hashes using zip2john, then cracking them with a wordlist. The first three passwords revealed a pattern involving the word chips, which helped build a targeted attack for the final archive. Using John rules transformed chip into chip!, revealing the last password and the final flag.

Lessons Learned

  • Filenames can contain direct operational hints.
  • When general cracking stalls, create a targeted candidate set.
  • Quote shell-special passwords like chip! during extraction.
